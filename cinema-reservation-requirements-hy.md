# Կինոյի ամրագրման հարթակ — Առաջադրանքի նկարագիր

## Ամփոփում

Կառուցել ամբողջական (end-to-end) Կինոյի ամրագրման հարթակ՝ Node.js-ի, Express-ի և MySQL-ի միջոցով։ Հավելվածը թույլ է տալիս այցելուներին դիտել ֆիլմերի ծրագիրը, ամրագրել նստատեղեր որևէ ցուցադրության համար և հետևել իրենց ամրագրումներին։ Աշխատակիցների հաշիվները ստանում են լրացուցիչ գործիքակազմ՝ կատալոգը կառավարելու և ցուցադրությունները պլանավորելու համար։

---

## Տեխնոլոգիական փաթեթ

- **Սերվեր:** Node.js, որի վրա աշխատում է Express.js (տարբերակ 5)
- **Հաճախորդ (client):** մաքուր HTML, CSS և JavaScript
- **Սխեմայի վավերացում:** Joi
- **Տվյալների պահպանում:** MySQL՝ **Sequelize** ORM-ի միջոցով (ներքևում՝ `mysql2` դրայվեր)
- **Նույնականացում / սեսիաներ:** JSON Web Tokens (`jsonwebtoken`)
- **Գաղտնաբառի հեշավորում:** MD5 (`md5`) կամ `crypto-js`
- **Ժամանակ և ամսաթվեր:** `moment`
- **Եզակի նույնացուցիչներ:** `uuid`
- **Կարգավորումներ:** `dotenv`
- **Ավտո-վերագործարկում (միայն մշակման ընթացքում):** `nodemon`

## Հաշիվները և նրանց հնարավորությունները

### Անդամ (սովորական հաշիվ)
- Դիտել ցուցադրությունների ծրագիրը և առկա ֆիլմերը
- Գրանցվել և մուտք գործել
- Ընտրել նստատեղեր և հաստատել ամրագրումը
- Դիտել իր ամրագրումների պատմությունը
- Տեսնել կատարված ամրագրման հաստատումը

### Աշխատակից (ընդլայնված հաշիվ)
- Ամեն ինչ, ինչ կարող է անդամը, գումարած՝
- Ստեղծել, թարմացնել և հեռացնել ֆիլմեր
- Պլանավորել ցուցադրություններ (օրական ոչ ավելի, քան 3-ը մեկ ֆիլմի համար)
- Դիտել համակարգի բոլոր ամրագրումները
- Կարդալ հիմնական վիճակագրությունը (ոչ պարտադիր)

---

## Պարտադիր ֆունկցիոնալություն

### 1. Նույնականացում՝ գրանցում, մուտք, լիազորում

- **Հաշվի ստեղծում**
  - Գրանցում էլ. փոստի և գաղտնաբառի հիման վրա
  - Էլ. փոստի ձևաչափը ստուգվում է Joi-ով
  - Գաղտնաբառը պետք է լինի առնվազն 6 նիշ
  - Մերժել կրկնվող օգտանուններ կամ էլ. փոստեր
  - Պահպանել գաղտնաբառի MD5 հեշը, ոչ թե բաց տեքստը

- **Նույնականացում (Authentication)**
  - Հաջող մուտքից հետո տրամադրել JWT
  - Թոքենը պահել հաճախորդի կողմում (localStorage կամ cookie)
  - Հաշվի առնել թոքենի ժամկետի ավարտը
  - Ապահովել ելքի (sign-out) ուղի, որը ջնջում է թոքենը

- **Հավատարմագրեր (Credentials)**
  - Թույլ տալ մուտք գործած հաշվին փոխել գաղտնաբառը

### 2. Կատալոգի կառավարում

- **Կատալոգի դիտում**
  - Ցուցադրել բոլոր առկա ֆիլմերը՝ էջ առ էջ (pagination), էջում 10-ական
  - **Որոնում ըստ վերնագրի (հիմնական պահանջ):** գտնել ֆիլմերը ըստ վերնագրի մուտքագրման ընթացքում
  - Յուրաքանչյուր ֆիլմ ներկայացնում է՝ վերնագիր, ժանր, տևողություն, համառոտագիր
  - Ցույց տալ էջերի համարները և Նախորդ / Հաջորդ կոճակները

- **Աշխատակցի կատալոգի գործիքներ**
  - Գրանցել նոր ֆիլմ՝ իր ամբողջական հատկանիշներով
  - Փոփոխել գոյություն ունեցող ֆիլմի մանրամասները
  - Հեռացնել ֆիլմ, բայց միայն եթե դրան կապված ամրագրումներ չկան
  - Ցուցադրել բոլոր ֆիլմերը՝ նույն որոնման և pagination-ի հնարավորություններով

### 3. Հանդիսատեսի կարծիքներ (հիմնական պահանջ — կիրառում է JOIN-ներ Sequelize-ի միջոցով)

Թույլ է տալիս անդամներին հրապարակել կարծիքներ այն ֆիլմերի մասին, որոնք իրականում դիտել են։

**Անդամի գործողություններ:**
- **Թողնել կարծիք**
  - Թույլատրվում է միայն այն ֆիլմերի համար, որոնք անդամն ամրագրել/դիտել է
  - Ազատ տեքստ՝ առավելագույնը 500 նիշ
  - 1–5 աստղ գնահատական (կարող եք դարձնել պարտադիր կամ ոչ պարտադիր)
  - Պայման՝ անդամը պետք է ունենա առնվազն մեկ հաստատված ամրագրում տվյալ ֆիլմի համար
  - Յուրաքանչյուր կարծիք ցույց է տալիս հեղինակի անունը և ամսաթիվը
- **Խմբագրել սեփական կարծիքը**
  - Հեղինակը կարող է խմբագրել հրապարակումից հետո 24 ժամվա ընթացքում
  - 24 ժամ անցնելուց հետո փակվում է
  - Անցած ժամանակը հաշվարկելու համար օգտագործել `moment`
- **Հեռացնել սեփական կարծիքը**
  - Հեղինակը կարող է ցանկացած պահի հեռացնել այն
  - Կամ փափուկ ջնջում (`status = 'deleted'`), կամ կոշտ ջնջում

**Կարծիքների ցուցադրում:**
- Ֆիլմի մանրամասների էջում ցույց տալ հաստատված կարծիքները
- Pagination՝ էջում 10 կարծիք
- Առաջարկել դասավորություն ըստ՝ նորագույն, հնագույն, ամենաբարձր, ամենացածր գնահատականի
- Տպել կարծիքների ընդհանուր քանակը
- Տպել ֆիլմի միջին գնահատականը՝ ստացված դրա բոլոր կարծիքներից
- Յուրաքանչյուր գրառում ցույց է տալիս՝ հեղինակ, աստղային գնահատական, տեքստ, հրապարակման ամսաթիվ

**Աշխատակցի գործողություններ:**
- **Կարծիքների մոդերացիա**
  - Հաստատել կամ մերժել սպասվող կարծիքները (երբ մոդերացիան միացված է)
  - Հեռացնել անպատշաճ կարծիքները
  - Ցուցադրել բոլոր կարծիքները՝ իրենց ֆիլմի հետ միասին (JOIN-ների միջոցով)
  - Զտել ցանկն ըստ կարգավիճակի (pending / approved / rejected)

**Տվյալների կանոններ:**
- Մեկ կարծիք՝ մեկ անդամից մեկ ֆիլմի համար — առանց կրկնությունների (ստուգումն իրականացրեք ձեր հավելվածի/Sequelize-ի տրամաբանության մեջ, օր.՝ ստուգել նախքան insert-ը)
- Կարծիքների աղյուսակը պետք է ունենա՝ id, user_id, film_id, rating, comment_text, status, created_at, updated_at

**Հարցումները, որ պետք է գրեք (Sequelize-ի associations / `include`-ի միջոցով):**
- Կարծիքները՝ միացված իրենց հեղինակին (JOIN users-ի վրա)
- Կարծիքները՝ միացված իրենց ֆիլմին (JOIN films-ի վրա)
- Հաստատել, որ անդամը դիտել է ֆիլմը (JOIN bookings-ի և showtimes-ի վրայով)

### 4. Ցուցադրությունների ժամանակացույց

- **Ժամանակացույցի ցուցադրում**
  - Ներկայացնել յուրաքանչյուր ֆիլմի համար առկա ամսաթվերն ու ժամերը
  - Օրական առավելագույնը 3 ցուցադրություն մեկ ֆիլմի համար
  - Նշել՝ արդյոք նստատեղեր մնացել են

- **Աշխատակցի ժամանակացույցի գործիքներ**
  - Ավելացնել ցուցադրություններ ընտրված ամսաթվերին
  - Նշանակել մեկ ֆիքսված տոմսի գին յուրաքանչյուր ցուցադրության համար

---

### 5. Նստատեղերի ընտրություն և ամրագրում

- **Ինտերակտիվ նստատեղերի ցանց**
  - Տող/սյունակ տեսքով տեսանելի դասավորություն
  - Կարգավիճակը փոխանցվում է գույնով՝
    - Ազատ (կանաչ)
    - Ընտրված (դեղին)
    - Զբաղված (կարմիր)
  - Թույլ տալ միանգամից մի քանի նստատեղ ընտրել
  - Բոլոր նստատեղերն ունեն նույն գինը

- **Ամրագրման ընթացք**
  - Իրական ժամանակում ստուգել նստատեղերի առկայությունը
  - Արգելափակել նույն նստատեղի կրկնակի ամրագրման ցանկացած փորձ
  - Հաշվել ընդհանուր գինը՝ ընտրված նստատեղերից
  - Անցնել հաստատման էջ
  - `uuid`-ով ստեղծել եզակի ամրագրման համար (reference)

- **Ամրագրման սահմանափակումներ**
  - Մեկ ամրագրման մեջ ոչ ավելի, քան 6 նստատեղ

### 6. Ամրագրումների պատմություն և կառավարում

- **Անդամի ամրագրումների պատմություն**
  - Օրինակ ձևակերպում՝ "Tom Ford booked seat 4/5 at 15:30 on 22 Aug 2024"
  - Ամսաթվերն ու ժամերը մաքուր ձևաչափել `moment`-ով
  - Ցույց տալ ֆիլմի վերնագիրը, ցուցադրության ժամանակը, նստատեղերը և ընդհանուր արժեքը
  - **Pagination (հիմնական պահանջ):** էջում 10 ամրագրում
  - Էջերի կառավարում (Նախորդ / Հաջորդ, էջերի համարներ)
  - Ցույց տալ ամրագրումների ընդհանուր քանակը

- **Ամրագրման գործողություններ**
  - Բացել ամրագրման մանրամասները
  - Չեղարկել ամրագրումը (ոչ պարտադիր)

---

### 7. Վերլուծություն և հաշվետվություններ (հիմնական պահանջ — կիրառում է JOIN-ներ)

Այստեղ դուք ցուցադրում եք տվյալների հետ բարդ աշխատանք՝ JOIN-ներով՝ Sequelize-ի associations-ի և `include`-ի միջոցով։

**Անդամի վահանակ:**
- **Իմ թվերը** (մուտք գործած անդամների համար)
  - Ամրագրումների ընդհանուր ծախսը (ողջ ընթացքում)
  - Ամրագրումների ընդհանուր քանակը
  - Նախընտրած ժանրը (ամենից շատ ամրագրված ժանրը)
  - Ամենից շատ դիտված ֆիլմը

**Աշխատակցի վահանակ — Ֆիլմերի վերլուծություն:**
- **Լավագույն ֆիլմերի հաշվետվություն** (JOIN)
  - Ֆիլմերը՝ իրենց ամրագրումների ընդհանուր քանակով
  - Յուրաքանչյուր ֆիլմից ստացված եկամուտը
  - Միջին ամրագրված նստատեղերը մեկ ցուցադրության համար
  - Դասավորված ըստ ժողովրդականության (առավելագույն ամրագրումները՝ առաջինը)

**Աշխատակցի վահանակ — Եկամտի վերլուծություն:**
- **Եկամուտ ըստ օրվա / շաբաթվա**
  - Եկամուտը՝ գումարված ըստ ամսաթվի
  - Ամրագրումների քանակը ըստ ամսաթվի
  - Գծապատկեր կամ աղյուսակ, որը ցույց է տալիս միտումը
  - Զտվող ըստ ամսաթվերի միջակայքի

**Աշխատակցի վահանակ — Անդամների վերլուծություն:**
- **Լավագույն անդամների հաշվետվություն** (JOIN)
  - Ամենաշատ ամրագրումներ ունեցող անդամները
  - Յուրաքանչյուր անդամի ընդհանուր ծախսը
  - Սյունակներ՝ օգտանուն, ամրագրումների քանակ, ընդհանուր ծախս
  - Միայն 5+ ամրագրում ունեցող անդամները որակավորվում են

**Ցուցադրությունների արդյունավետություն:**
- **Ցուցադրությունների վիճակագրություն** (մի քանի JOIN)
  - Յուրաքանչյուր ցուցադրություն՝ նշված իր ֆիլմի վերնագրով
  - Ամրագրված նստատեղեր՝ ընդդեմ առկա նստատեղերի
  - Լրացվածության տոկոս (ամրագրված / առկա × 100)
  - Եկամուտ մեկ ցուցադրության համար
  - Դասավորված ըստ լրացվածության տոկոսի

## Լրացուցիչ ընտրովի հնարավորություններ

Ընտրեք 1–2-ը, եթե ժամանակ կա՝

### Ա. Ֆիլմերի պաստառներ (հղումների միջոցով)
- Ֆիլմի գրառման մեջ պահել պաստառի պատկերի URL — առանց ֆայլերի վերբեռնման
- Կատալոգի ցանկում ցույց տալ պաստառները՝ բեռնելով դրանք իրենց URL-ից
- Եթե URL նշված չէ, ցույց տալ placeholder

### Բ. Ավելի հարուստ զտում
- Զտել ֆիլմերն ըստ ժանրի (բացվող ցանկ)
- Զտել ըստ տևողության խմբի (կարճ / միջին / երկար)
- Համատեղել զտիչները որոնման հետ
- Դասավորել ըստ վերնագրի (Ա–Ֆ, Ֆ–Ա) կամ ավելացման ամսաթվի

### Գ. Հաշվի պրոֆիլ
- Պրոֆիլի էջ
- Դիտել և խմբագրել անձնական տվյալները
- Թարմացնել էլ. փոստն ու հեռախոսը
- Փոխել գաղտնաբառը՝ հաստատումով
- Ցույց տալ հաշվի վիճակագրությունը (ամրագրումների քանակ, ընդհանուր ծախս)

### Դ. Հաստատման բարելավում
- Մանրամասն տեսք հաջող ամրագրումից հետո
- Ամրագրման համարը (UUID) կոդավորել որպես QR կոդ
- Ամսաթիվ/ժամ ձևաչափել `moment`-ով
- Տպելու համար պատրաստ տոմս
- Ամրագրման մանրամասները կիսվելու տարբերակ

## Տվյալների բազայի նախագիծ

### Աղյուսակներ

1. **users**
   - id — PK, AUTO_INCREMENT
   - username — VARCHAR, UNIQUE
   - email — VARCHAR, UNIQUE
   - password — VARCHAR (MD5 հեշ)
   - full_name — VARCHAR
   - role — ENUM('user', 'admin')
   - created_at — DATETIME

   Նշում. JWT-ները պահվում են հաճախորդի կողմում, ոչ թե այս աղյուսակում։

2. **films**
   - id — PK, AUTO_INCREMENT
   - title — VARCHAR
   - description — TEXT
   - genre — VARCHAR (օր.՝ Action, Comedy, Drama, Horror, Sci-Fi)
   - duration — INT (րոպեներ)
   - created_at — DATETIME

3. **showtimes**
   - id — PK, AUTO_INCREMENT
   - film_id — FK → films.id
   - show_date — DATE
   - show_time — TIME
   - price — DECIMAL
   - total_seats — INT (օր.՝ 50)
   - available_seats — INT
   - created_at — DATETIME

4. **bookings**
   - id — PK, AUTO_INCREMENT
   - user_id — FK → users.id
   - showtime_id — FK → showtimes.id
   - seats — VARCHAR (օր.՝ "A1,A2,A3")
   - total_price — DECIMAL
   - booking_reference — VARCHAR, UNIQUE (uuid-ից)
   - booking_date — DATETIME
   - status — ENUM('confirmed', 'cancelled')

5. **comments**
   - id — PK, AUTO_INCREMENT
   - user_id — FK → users.id
   - film_id — FK → films.id
   - rating — INT, 1–5 (NULL թույլատրելի կամ NOT NULL, ձեր ընտրությամբ)
   - comment_text — TEXT (≤ 500 նիշ)
   - status — ENUM('pending', 'approved', 'rejected', 'deleted')
   - created_at — DATETIME
   - updated_at — DATETIME

### Կապեր (ձեր JOIN-ների հիմքը)

- Մեկ ֆիլմ → շատ ցուցադրություններ
- Մեկ ցուցադրություն → շատ ամրագրումներ
- Մեկ անդամ → շատ ամրագրումներ
- Մեկ անդամ → շատ կարծիքներ
- Մեկ ֆիլմ → շատ կարծիքներ
- Շատ ամրագրումներ → մեկ ցուցադրություն → մեկ ֆիլմ (լուծվում է JOIN-ով)
- Շատ կարծիքներ → մեկ անդամ (JOIN users՝ հեղինակի անունը ստանալու համար)
- Շատ կարծիքներ → մեկ ֆիլմ (JOIN films՝ ֆիլմի մանրամասները ստանալու համար)

## Որոնում և Pagination (հիմնական պահանջ)

### Ինչու են դրանք կարևոր
1. **Որոնումը** թույլ է տալիս անդամներին արագ հասնել ֆիլմին՝ առանց ամբողջ կատալոգը ոլորելու։
2. **Pagination-ը** էջերը պահում է արագ, երբ կան շատ ֆիլմեր կամ ամրագրումներ։

### Որտեղ են պարտադիր

**Pagination-ը պարտադիր է՝**
- Կատալոգի ցանկում (էջում 10)
- Անդամի ամրագրումների պատմության մեջ (էջում 10)
- Աշխատակից՝ ֆիլմերի ամբողջական ցանկ (էջում 10)
- Աշխատակից՝ ամրագրումների ամբողջական ցանկ (էջում 20) — ոչ պարտադիր

**Որոնումը պարտադիր է՝**
- Կատալոգի ցանկում (ըստ վերնագրի)
- Աշխատակցի ֆիլմերի կառավարման մեջ (ֆիլմերի որոնում)

### Ինչ պետք է ներառի ձեր pagination-ը
- Ընթացիկ էջի համարը
- Էջերի ընդհանուր քանակը
- Նախորդ կոճակ, անջատված 1-ին էջում
- Հաջորդ կոճակ, անջատված վերջին էջում
- Էջերի համարների հղումներ (1, 2, 3, 4, 5, …)
- "X–Y of Z results" ցուցիչ

### Ինչ պետք է ներառի ձեր որոնումը
- Որոնման դաշտ էջի վերևում
- Որոնում ուղարկելիս (submit), կամ կենդանի որոնում կոճակով
- "No results found" վիճակ, երբ ոչինչ չի համընկնում
- Որոնումը մաքրելու կոճակ, որը վերականգնում է տեսքը
- Որոնում, որը համագործակցում է pagination-ի հետ (արդյունքները նույնպես էջավորվում են)

## JWT-ի հոսքը

### Ինչպես են թոքենները շարժվում հավելվածի միջով
1. **Գրանցում:** գաղտնաբառը հեշավորել MD5-ով, պահել անդամին, թոքեն դեռ չստեղծել։
2. **Մուտք:** ստուգել էլ. փոստ/գաղտնաբառը; եթե ճիշտ է, ստեղծել JWT՝ անդամի տվյալներով և վերադարձնել այն (մարմնում կամ cookie-ում)։
3. **Պաշտպանված ուղիներ:** հաճախորդը յուրաքանչյուր հարցմանը կցում է թոքենը (header կամ cookie); middleware-ը վավերացնում է այն; վավեր → անցնել, անվավեր → ուղղորդել դեպի մուտք։

### Օրինակ payload
```javascript
{
  id: 1,
  email: "user@example.com",
  role: "user"
}
```

### Որտեղ պահել թոքենը
- **Տարբերակ 1:** localStorage (ավելի պարզ)
- **Տարբերակ 2:** HTTP-only cookie (ավելի անվտանգ)

## Ինժեներական պահանջներ

### Վավերացում և սխալներ
- **Վավերացում**
  - Վավերացնել ձևի յուրաքանչյուր մուտքագրում Joi-ով
  - Ստուգել էլ. փոստի ձևը, գաղտնաբառի երկարությունը և պարտադիր դաշտերը
  - Վերադարձնել ընթեռնելի սխալի հաղորդագրություններ
- **Սխալների մշակում**
  - Հատուկ 404 էջ
  - Տվյալների բազայի ձախողումների սահուն մշակում
  - Հստակ, օգտակար հաղորդագրություններ օգտատիրոջ համար

### Անվտանգություն
- Տվյալների բազայի հետ բոլոր աշխատանքի համար օգտագործել Sequelize (լռելյայն պարամետրացված)՝ SQL injection-ը կանխելու համար
- Պահել միայն գաղտնաբառների MD5 հեշը
- Նույնականացնել `jsonwebtoken`-ով
- Պաշտպանել ուղիները JWT middleware-ով, որը վավերացնում է թոքենը
- Վավերացնել բոլոր մուտքագրումները Joi-ով
- Գաղտնիքները (JWT secret) պահել `.env`-ում
- Սահմանել թոքենի կյանքի տևողություն (օր.՝ 24 ժամ)

### Կոդի որակ
- Հետևել MVC օրինաչափությանը (models, views, controllers)
- Կոդը բաժանել առանձին թղթապանակների
- Տալ իմաստալից անվանումներ
- Մեկնաբանել բարդ հատվածները
- Նախագիծը վարել Git-ով

## Ինչ հանձնել

1. **Կոդ**
   - Ամբողջական Node.js հավելվածը
   - Կոկիկ թղթապանակային կառուցվածք
   - Մեկնաբանություններ այնտեղ, որտեղ դրանք օգնում են

2. **Տվյալների բազա**
   - MySQL բազա՝ բոլոր աղյուսակներով
   - Սկզբնական (seed) տվյալներ (առնվազն 3 ֆիլմ, մի քանի անդամ, որոշ ամրագրումներ)

3. **Փաստաթղթեր**
   - README.md, որն ընդգրկում է՝
     - Տեղադրման և գործարկման քայլերը
     - Տվյալների բազայի կարգավորումը
     - Թեստային հավատարմագրեր անդամի և աշխատակցի հաշիվների համար

## Գնահատում

- **Ֆունկցիոնալություն (25%):** բոլոր հիմնական հնարավորությունները աշխատում են
- **Տվյալների մատչում — JOIN-ներ Sequelize-ով (25%):** վերլուծությունն ու կարծիքները կառուցված են ամուր associations/`include` հարցումների վրա
- **Որոնում և Pagination (10%):** ճիշտ կիրառված՝ որտեղ պահանջվում է
- **Կոդի կազմակերպում (15%):** մաքուր կառուցվածք, MVC
- **Տվյալների բազայի նախագիծ (15%):** հիմնավոր աղյուսակներ և կապեր
- **Ինտերֆեյս (5%):** հեշտ օգտագործելի և նավարկելի
- **Վավերացում և անվտանգություն (5%):** մուտքագրման վավերացում և գաղտնաբառի հեշավորում

## Կարգավորման ուղեցույց

1. Տեղադրեք Node.js և MySQL։
2. Ստեղծեք նոր նախագծի թղթապանակ։
3. `npm init -y`՝ package.json ստեղծելու համար։
4. Տեղադրեք կախվածությունները՝
   ```bash
   npm install express sequelize mysql2 joi jsonwebtoken crypto-js md5 moment uuid dotenv morgan http-errors debug lodash
   ```
5. Ավելացրեք nodemon՝ մշակման համար՝
   ```bash
   npm install --save-dev nodemon
   ```
6. Կարգավորեք package.json-ի scripts-ը՝
   ```json
   "scripts": {
     "start": "node app.js",
     "dev": "nodemon app.js"
   }
   ```
7. Ստեղծեք `.env`՝
   ```
   DB_HOST=localhost
   DB_USER=root
   DB_PASSWORD=your_password
   DB_NAME=cinema_reservation
   JWT_SECRET=your_secret_key_here
   PORT=3000
   ```
8. Ստեղծեք MySQL բազան և դրա աղյուսակները։
9. Կազմեք թղթապանակները (տես վերևի կառուցվածքը)։
10. Սկսեք գրանցում/մուտքից և JWT-ից։
11. Ավելացրեք հնարավորությունները մեկ առ մեկ։
12. Թեստավորեք յուրաքանչյուր հնարավորություն հաջորդից առաջ։
13. Գործարկեք այն՝ `npm run dev`։
